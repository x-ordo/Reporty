# PR: 프로젝트 구조 리팩터링 및 초기 개선

## 설명

프로젝트의 유지보수성, 보안, 확장성을 강화하기 위한 대규모 리팩터링 및 초기 개선 사항을 포함하는 PR입니다.

이 PR은 다음 작업을 포함합니다:

-   **프로젝트 구조 표준화**: 프런트엔드 소스를 `src`로, 백엔드를 `backend`로 명확히 분리하고 문서화를 위해 `docs` 디렉터리를 도입했습니다.
-   **보안 강화**: 파트너 키 인증 로직을 `bcrypt` 기반의 강력한 해싱으로 교체하여 보안 취약점을 해결했습니다.
-   **아키텍처 일관성 확보**: 프런트엔드의 PDF 생성 로직을 백엔드의 비동기 API를 사용하도록 통일했습니다.
-   **테스트 기반 마련**: `vitest`를 도입하고 핵심 보안 로직에 대한 초기 테스트를 작성했습니다.

## 관련된 Issue

-   **Partially addresses:** #3 (사용자 인증 시스템 고도화) - *Note: bcrypt 도입으로 인증 시스템의 기반을 강화했습니다.*

## 변경 사항 요약

-   `bcrypt` 의존성 추가
-   `crypto.ts`: `bcrypt`를 사용한 해싱 및 비교 함수 추가
-   `auth.ts`: 새로운 `bcrypt` 비교 로직을 사용하도록 인증 함수 변경
-   `seed.mjs`: 시드 스크립트가 새로운 해싱 로직을 사용하도록 업데이트
-   `DefenseReport.tsx`: `window.print()` 대신 백엔드 API를 호출하여 PDF 생성 요청
-   `vitest`, `dotenv-cli` 등 테스트 관련 의존성 추가
-   `vitest.config.ts` 및 `package.json`에 테스트 스크립트 추가
-   프로젝트 전반의 파일 및 폴더 구조 변경

## 리뷰어에게

-   `backend/.env` 파일에 테스트용 더미 데이터가 포함되어 있습니다. 실제 환경에서는 이 파일을 사용하지 않도록 `.gitignore`에 추가하는 것을 고려해야 합니다.
-   테스트 실행(`npm test`)은 현재 환경 변수 문제로 실패합니다. 이 문제를 해결하는 것이 최우선 후속 과제입니다.
